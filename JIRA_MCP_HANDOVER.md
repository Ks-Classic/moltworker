# 🚀 OpenClaw × Jira 統合プロジェクト 引継ぎ書

このドキュメントは、これまでの検討文脈を新しいAIセッション（Serena等）に引き継ぐための要件定義・アーキテクチャ設計書です。

## 🎯 プロジェクトの目的
OpenClaw (Discord Bot / AI Agent) が、Jiraを「CRM兼タスク管理の司令塔」としてシームレスに操作できるようにする。
これまでLarkの表（スプレッドシート形式）で管理していた全情報をJiraに移行し、AIがそれを管理・集計・更新できるように `moltworker` にツールを実装する。

## 🏗️ Jira側のアーキテクチャ設計（決定済）
Jiraの1つのプロジェクト内に、以下の「課題タイプ (Issue Types)」を設定して運用する。
これにより、AIは「JQL」を用いて全フェーズを横断検索できるようになる。

1. **🤝 商談 (Opportunity)**: [ステータス: 提案中/受注/失注]
2. **🚀 案件 (Project)**: [ステータス: 開発中/確認中/完了] (※商談から移行)
3. **💡 アイディア (Idea)**: [ステータス: 起案/採用]
4. **✅ タスク (Task)**: 各案件に紐づく実作業

**カスタムフィールド（Jira側で用意）**:
- `金額` (Number)
- `期日` (Date)
- `顧客名` (Text)

※ 商談のやりとりやメモは、すべて「Jiraのコメント機能」と「添付ファイル機能」を使用する設計とする。

## 💻 開発要件（moltworkerへの実装）
`moltworker` に以下のいずれかの形でJira連携機能を組み込むこと。
現状のコードベース（MCPを利用しているか、独自のTools関数形式か）を確認し、最適なアプローチを提案してから実装に入ること。

### 推奨アプローチ
**Jira MCP サーバーの統合**
現在ローカルで動作する `mcp-jira` サーバーと同等の仕組みをbotに接続する。これにより、AIは `manage_jira_issue` や `analyze_jira_issues` などの強力なツール群を即座に利用可能になる。

**代替アプローチ**
**Jira REST API v3 の直接実装**
JiraのAPIトークンを用いて、`moltworker` 内に以下の必須ツールを関数として実装する。
1. JQLによる課題検索 (`searchIssues`)
2. 課題の作成/更新 (`createIssue`, `transitionIssue`)
3. コメントの追加 (`addComment`)
